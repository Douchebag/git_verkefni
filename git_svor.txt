Git spurningar og svor

1. Hvað gera eftirfarandi Linux skipanir?
	cd breytir current directory i home directory
	cd ~ breytir current directory i root directory
	ls prentar a skjainn lista yfir directories i directory-inu sem vid erum i
	pwd prentar directory-id sem vid erum staddir i
	mkdir byr til nytt directory

2. Hvað gera eftirfarandi git skipanir og hvers vegna eru þær gagnlegar?
	git clone clone-ar repo inni nytt directory
	git log birtir lista yfir commit skipunum
	git status synir hversu langt local repo-id er a undan master branchinu
	git diff synir mismun a commits 
	git checkout notad til ad breyta um branch eda endurheimta hluti

3. Hver er munurinn á eftirfarandi git skipunum:
	a) git diff synir mismun a commits
	b) git diff --staged synir breytingarnar sem hafa verid settar upp fyrir naesta commit
	c) git diff commit1 commit2 synir muninn a tveimur akvednum commits

4. 	Hvað er útgáfustýring (e.version control)? 
	Utgafustyring er umsjon breytingar a skjolum, forritum, storum vefsidum og odrum sofnudum upplysingum. 

5.	Hverjir eru helstu kostir við að nota GIT?
	Thad er back-up af ollu sem thu hefur verid ad vinna a. Thu getur alltaf farid i fyrra version af thvi sem thu ert ad vinna a og nad i thad aftur. Git getur verid notad til thess ad margir geta verid ad vinna a thvi sama. 

6.	Hversu oft telur þú að eigi að gera commit í verkefni, rökstuddu?
	Thad aetti ad vera gert commit eins oft og ad hann sem er ad vinna i verkefninu telur thorf fyrir. 

7.	Hvað er "Working directory"?" "Staging area"?" og "Repository" í GIT?
	Working directory er directory-id sem er verid ad vinna inn i.
	Staging area er skra sem inniheldur upplysingar um hvad mun fara inni naesta commit-id thitt.
	Repository er geymsla fyrir allar skrar o.fl. sem thu ert ad vinna med i git.

8.	Hvenær er sniðugt að nota greinar (branches)? 
	Til daemis thegar margir eru ad vinna a einu verkefni er gott ad nota greinar. Thegar thu villt prufa eitthvad sem thu ert ekki viss um ad thu viljir endilega hafa i master-inu.
